Go, a linguagem do futuro?

A linguagem de programação Go, também pode ser lembrada como “Golang do google”, está em crescimento nos últimos anos, com novos modelos se adequando com a programação moderna, especialmente na nuvem, e o crescimento do Go aconteceu por ser considerada uma linguagem leve e com o código aberto. Outro campo onde a linguagem se destaca é o da ciência de dados, com a capacidade de passar do “laptop do analista para a produção total”.
É uma linguagem de engenharia, e que beneficia os desenvolvedores de diversas maneiras, por exemplo, o garbage collection, que é a concorrência ativa que reduz a necessidade de código para se lidar com vazamentos de memória. A linguagem Go também possui diversos outros recursos que se encaixam nas arquiteturas de micro serviços e na ciência de dados.
Levando isso em consideração, o go está sendo adotado por diversas empresas, atualmente, uma API para o tensorflow foi colocada, e produtos como o pachyderm, estão sendo criados na Linguagem go.
The Stack Overflow Survey (2017), mostra alguns sinais da popularidade do Go, a pesquisa deles mostra que, ela possui dados coletados de 64.000 desenvolvedores, e procura descobrir suas preferencias, para que se torne cada vez mais querido.
Algumas linguagens de programação, cresceram colaborativamente durante um período, ou, foram projetadas em uma diferente era, de computação com diferentes necessidades, o Go é uma linguagem explicitamente projetada, que foi direcionada a resolver problemas com linguagens e ferramentas, enquanto aproveita nativamente arquiteturas de hardware modernas.
O Go, é uma linguagem fácil, e no mundo da TI, o software complexo, em uma grande escala, e é escrita por equipes de desenvolvedores, e geralmente possuem diferentes níveis de habilidades, desde menos experientes até mais experientes.
Além disso, possui um idioma, que incentiva a legibilidade e compreensão extremamente útil, A mistura de tipagem duck (via interfaces) e features de conveniência, como “:=” para declarações de variáveis curtas, dão a sensação de uma linguagem dinamicamente tipada, mantendo também os benefícios de uma tipagem forte.
O garbage, collection nativo do Go, que retira a necessidade de programadores produzirem seu próprio gerenciamento de memória, ajudando a evitar problemas como:
1 - Esperar o gerenciamento de memória serem feitos.
2 - O gerenciamento de memória requer diferentes rotinas para diferentes cores de processamento, o manual de contabilizar cada configuração pode aumentar significamente o risco de vazamentos de memória.
A concorrência do Go é importante para aplicações de rede que vivem e morrem em concorrência. Das APIs aos servidores web e frameworks de aplicativos web, os projetos envolvendo Go tendem a se concentrar em redes, funções distribuídas e/ou serviços para os quais os “goroutines” e canais do Go sejam apropriados.
